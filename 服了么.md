# 《赢了么》v0.1 核心战斗系统草案

## 1. 项目定位

### 游戏类型

单机卡牌构筑 / 回合制对战 / 轻 roguelike

### 核心卖点

表面是现代互联网抽象嘴炮对决。
底层是一个围绕 **体面、气势、舆论、破防** 四资源展开的策略卡牌系统。

### 设计目标

* **内容抽象**：牌名、角色、演出、文案要有现代赢学味道
* **机制硬核**：资源博弈、响应反制、状态克制、构筑流派要成立
* **开发可控**：先做白盒，不依赖重美术，不依赖联网

### 首版目标

做出一个 **PC 浏览器可玩的 Unity WebGL Demo**

* 1 名玩家角色
* 3 类普通敌人
* 1 个 Boss
* 40 张测试牌
* 2 个场地
* 单局 10~15 分钟

---

# 2. 核心资源系统

游戏不是普通打血，而是四资源博弈。

## 2.1 体面值 Face

相当于生命值。
归零则失败。

### 初始值

* 玩家：30
* 敌人：20~35，按模板调整

### 说明

不是肉体生命，而是“场面站不住了”。
大多数进攻牌最终会作用到体面值，但不应只有这一种取胜手段。

---

## 2.2 气势 Momentum

相当于行动资源 / mana。

### 初始规则

* 每回合开始获得 3 点气势
* 最大气势上限 10
* 未使用气势默认不保留
* 某些状态/遗物/牌可修改规则

### 设计目的

控制出牌节奏，形成低费运营、中费反制、高费终结的层次。

---

## 2.3 舆论 Public Opinion

这是本作的核心特色资源，表示场面支持度与话语主导权。

### 取值范围

-7 到 +7
其中：

* 正数：玩家占优
* 负数：敌人占优
* 0：僵持

### 作用

很多牌会根据舆论差获得额外效果：

* 舆论领先时，某些终结牌更强
* 舆论落后时，某些逆风牌更强
* 舆论可作为部分牌的费用或判定条件

### 设计目的

让战斗不只是“掉血”，而是有“控场”和“造势”。

---

## 2.4 破防值 Tilt

表示情绪失控程度。

### 基础规则

* 默认范围 0~10
* 达到 10 时触发 **失态**
* 失态后：

  * 立刻受到 4 点体面伤害
  * 舆论朝对方偏移 2 点
  * 破防值清零

### 作用

* 作为另一条胜利推进路径
* 支撑“情绪压制流”“破防流”“逆风爆发流”
* 让防守不仅仅是挡伤害，还包括情绪管理

---

# 3. 回合流程

## 3.1 标准回合结构

### 阶段 A：回合开始

* 重置本回合气势为基础值
* 抽 5 张牌（首回合可抽 5，以后回合抽 1~2，可配置）
* 处理“回合开始”触发
* 处理持续状态

### 阶段 B：出牌阶段

当前行动方可：

* 使用手牌
* 消耗气势
* 触发连锁
* 视情况发动有限响应

### 阶段 C：结算阶段

* 检查牌效结算
* 检查舆论变化
* 检查破防与失态
* 检查击败条件

### 阶段 D：回合结束

* 弃牌到手牌上限
* 清理“本回合”类状态
* 进入对手回合

---

## 3.2 响应机制（重点）

本作不建议一上来做成万智牌级别的全时点响应。
建议做 **轻响应机制**。

### 规则

* 只有带有 **【回应】** 标签的牌，能在对手行动时使用
* 每方每回合最多触发 1 次回应
* 回应牌通常费用较低，但条件较苛刻

### 目的

* 保留“舌战对招”的味道
* 控制复杂度
* 防止流程过度打断

### 示例

对方打出【数据打脸】后，玩家可以回应：

* 【质疑来源】：使其伤害 -3
* 【断章取义】：改为只影响舆论
* 【顾左右而言他】：取消其额外效果

# 4. 关键词系统

关键词必须少而硬，便于扩展。

## 4.1 基础关键词

### 【回应】

可在对手回合使用。
每回合每方默认仅可使用 1 次回应牌。

### 【转进】

改变牌的结算目标或资源维度。
例如把“体面伤害”改为“舆论变化”。

### 【定性】

给目标附加标签状态。
例如“双标”“嘴硬”“急了”。

### 【求锤】

若对方本回合未使用回应，则追加效果。

### 【失态】

当破防达到阈值触发的惩罚事件。

### 【带节奏】

直接改变舆论值。

### 【嘴硬】

当自身体面较低时，额外强化。

### 【翻旧账】

对已带状态的目标触发附加收益。

---

# 5. 状态系统

状态是深度来源之一。

## 5.1 核心状态

### 急了

* 持续 1 回合
* 使用攻击类牌时额外 +1 破防
* 使用防御类牌时效果 -1

### 双标

* 持续 2 回合
* 每次打出相同类别牌时，额外损失 1 点舆论

### 嘴硬

* 持续 2 回合
* 体面低于 10 时，攻击牌 +2
* 但无法回复体面

### 失语

* 持续 1 回合
* 下一张【回应】牌无法使用

### 主叙事

* 场面状态
* 持有方的【带节奏】类牌额外 +1 效果

### 已挂标签

* 被【定性】成功命中
* 某些“翻旧账”类牌可读此状态并增益

# 6. 牌的结构分类

不要只分攻击/防御。
建议六大类。

## 6.1 立论牌

建立框架、争夺节奏、获得资源。

例：

* 获得气势
* 获得主叙事
* 拉舆论
* 降低自己破防

---

## 6.2 论证牌

稳定推进主要资源差距。

例：

* 打体面
* 打破防
* 消耗对方气势
* 制造卡差

---

## 6.3 反制牌

回应、打断、抵消、修改结算。

例：

* 质疑来源
* 断章取义
* 你先别急
* 逻辑跳跃

---

## 6.4 定性牌

给对方挂标签、限制行动。

例：

* 双标警告
* 急了急了
* 你这外行
* 选择性失明

---

## 6.5 转进牌

改变战斗维度。

例：

* 顾左右而言他
* 经典转进
* 我们讨论的不是这个
* 先不谈这个

---

## 6.6 终结牌

高费强牌，需要前置条件。

例：

* 宣布胜利
* 拉黑退场
* 赢麻了
* 大家都懂

---

# 7. 初始流派设计

v0.1 先做 4 流派，别太多。

## 7.1 舆论流

### 核心

通过控制舆论获得牌效强化，最后用高条件终结牌收尾。

### 特征

* 控场
* 节奏稳定
* 中后期强

### 代表机制

* 带节奏
* 主叙事
* 舆论差判定

---

## 7.2 破防流

### 核心

通过叠破防逼对手失态，造成连锁崩盘。

### 特征

* 折磨
* 状态压制
* 不依赖高面板伤害

### 代表机制

* 急了
* 双标
* 失态触发
* 翻旧账

---

## 7.3 转进流

### 核心

改变结算资源与目标，打断敌方计划。

### 特征

* 技巧型
* 上限高
* 操作感强

### 代表机制

* 转进
* 回应
* 目标重定向
* 条件改写

---

## 7.4 嘴硬流

### 核心

低体面时强化，通过残血反打完成逆转。

### 特征

* 高风险高收益
* 爆发强
* 适合戏剧性演出

### 代表机制

* 嘴硬
* 残血加成
* 强终结

---

* 所有【回应】牌费用 -1，但最低为 0

# 11. 胜负与节奏目标

## 胜利条件

* 对方体面归零
* 或通过失态连锁把对方拖死

## 单局节奏目标

* 普通战斗：3~6 回合
* Boss 战：6~10 回合

## 玩家体验目标

前 2 回合：试探、拉资源
中盘：形成节奏差、压破防、抢舆论
后盘：根据流派决定终结方式

# 12. 白盒平衡原则

AI 开发第一版时必须遵守：

### 原则 1

低费牌主要负责：

* 过牌
* 小幅资源波动
* 挂状态
* 回应

### 原则 2

中费牌主要负责：

* 核心推进
* 联动
* 转进
* 稳定收益

### 原则 3

高费牌必须有前置条件，不能裸强

### 原则 4

所有“抽象文案牌”都必须有明确机制定位，禁止做纯梗牌

### 原则 5

先验证 3 个问题：

* 出牌是否有取舍
* 反制是否有存在感
* 舆论/破防是否真的参与决策

# 13. Unity 数据结构草案

## 13.1 CardData

```csharp
public enum CardType
{
    Thesis,      // 立论
    Argument,    // 论证
    Counter,     // 反制
    Label,       // 定性
    Redirect,    // 转进
    Finisher     // 终结

public enum TargetType
{
    Self,
    Enemy,
    None
}

public enum Keyword
{
    Response,
    Redirect,
    Label,
    DemandProof,
    MomentumShift,
    Stubborn,
    BringUpPast
}

[System.Serializable]
public class CardEffectData
{
    public string effectId;       // 如 "DealFaceDamage", "GainOpinion", "ApplyStatus"
    public int value;
    public string statusId;
    public string extraParam;
}

[CreateAssetMenu(menuName = "WinGame/Card")]
public class CardData : ScriptableObject
{
    public string cardId;
    public string cardName;
    public int cost;
    public CardType cardType;
    public TargetType targetType;
    public List<Keyword> keywords;
    public List<CardEffectData> effects;
    public string description;
}
```
